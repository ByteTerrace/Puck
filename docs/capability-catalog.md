# Puck capability catalog

The master inventory of everything Puck can do today. Puck is an
"everything-as-data" game engine — a dumb terminal with extra steps: one
versioned JSON document describes a run, and the engine's job is to render,
composite, validate, and replay it deterministically on whichever GPU backend
is available. There is no real gameplay yet beyond the overworld demo; the
value is the capability substrate below it, which this document catalogs.
(The overworld demo itself is **GREENFIELD** — a playground: its gameplay is
verified by RUNNING it, not gated, and the `Puck.Post` gates cited throughout
this catalog cover the ENGINE contract, not the demo. A demo feature is not
promoted into Post unless the user explicitly asks — see
[agent-guide.md](agent-guide.md#anti-calcification-doctrine) rule 5.)

Each entry says **what it is**, its **status**, **how to invoke it**, and
**where it lives**. Companion docs: [project-map.md](project-map.md) (which
project does what) and [agent-guide.md](agent-guide.md) (how to verify and
what will bite you).

**Status legend**

| Mark | Meaning |
|------|---------|
| ✅ | Shipped and verified on real hardware (RTX 4070 unless noted) |
| 🔶 | Shipped; partially verified, or verified only in one configuration |
| 🧪 | Designed-for: the seams exist, the end-to-end path is not built/tested |
| ⛔ | Known hard limit — documented, not a bug to fix |

## 1. Rendering core

| Capability | Status | Notes |
|---|---|---|
| SDF virtual machine | ✅ | Scenes are programs for a small instruction-stream VM, marched in compute shaders. 15 primitives (box, capsule, sphere, torus, cylinder, plane, ellipsoid, round cone, screen slab, vesica lens, and the **2D-primitive family** — rounded rectangle, regular polygon, star, trapezoid, ellipse — exact IQ 2D SDFs lifted to a 3D solid by `SdfLift.Revolve` (lathe around Y) or `.Extrude` (prism along Z), each earning a real cull bound instead of an approximation), 10 blends (union/intersection/subtraction + their smooth round-fillet forms + their chamfered 45°-bevel forms — `ChamferUnion`/`ChamferIntersection`/`ChamferSubtraction`, the mechanical/CAD counterpart to the smooth forms, sharing the same Data1.x blend-radius lane as a bevel size — + xor, with per-op material ownership — a carve wears the carving shape's material; unlike smooth-min's 1-Lipschitz fillet, a chamfer bevel's gradient reaches √2 at a FLAT/near-parallel seam (exactly 1 at a perpendicular one, → 0 at an acute one), so `AnalyzeLipschitz` bakes a conservative √2 step clamp whenever one is used, and a chamfer-free program is unaffected. **The accumulator rule:** a blend composes against every shape emitted before it — `ResetPoint` resets the point, not the running distance — so union and subtraction are local, but an intersection returns its candidate everywhere outside its own shape and annihilates earlier geometry it does not overlap. Author intersections first; an instance carrying one is unmaskable), point transforms (translate/rotate/scale/repeat/symmetry), warps (twistY, bendX/Y/Z, elongate, log-spherical Droste shells, a sine cross-coupled domain warp), field ops (onion, dilate, a sine-product surface-relief displace), and derived constants baked host-side by the builder (programs build once; shapes evaluate millions of times per frame). C# half in `src/Puck.SdfVm` (`SdfProgram`, `SdfProgramBuilder`); GPU half in the `sdf-vm` HLSL — the two ISAs must stay in sync. Gates: Post `world`, `world-menagerie` (new shapes + materials, incl. the vesica), `world-warp` (warps/field ops/blends), `world-log-sphere` (the Droste warp), `world-2d-family` (the 2D-primitive family, both lift modes, cross-backend), `world-chamfer` (the beveled blends, cross-backend parity). The final 8-bit store is **dithered** per-pixel (`sdfR2Dither`, an integer R2 / plastic-number value, bit-identical cross-backend so parity holds at maxΔ1), so smooth gradients — the sky ramp, exponential distance fog — break banding into blue-ish high-frequency noise instead of stepping. |
| Scoped field accumulator (PushField/PopField) | ✅ | `PushField(compose, smooth)`/`PopField` (opcodes 27/28, landed 2026-07-08) let an accumulator-reading op (`Intersection`/`Xor`/`Onion`/`Dilate`/`Displace`) compose against a FRESH `SDF_FAR_DISTANCE`-seeded scope instead of the whole flat field emitted before it — closing the "one-per-document field op" / "an intersection wipes every earlier shape" hole. Depth 1 (a validator rule, not an ISA limit — a `#define` raises the cap later without re-gating the packed layout). `AnalyzeLipschitz` folds a scope as a tree fold (a Union-compose POP over a warp-free subject bakes `stepScale` exactly `1.0f`; a chamfer-compose POP pins the same `√2` factor as a `ChamferUnion`). Landed consumers: `SceneObject.Dilate`/`Onion` (`src/Puck.Scene`) now scope PER OBJECT, not just object zero; the creator (and its `CompanionRenderer` clone) scopes each Pass-2 composition GROUP and each Pass-1 shape's onion — the depth-1 lesson: scoping is at GROUP/object granularity, not per-member-blend (a second nesting level this ISA doesn't carry). A scoped instance's cull bound inflates by its scoped field-op reach (`MaxScopedFieldReach`) so it stays maskable instead of falling back to `UnmaskableBoundRadius`. Gates: Post `world-scope` (cross-backend: a scoped intersection/smooth-union+onion/subtraction each compose far-neutrally onto an intact floor) + `sdf-lipschitz` (the tree-fold bake cases), full battery green. Open: run-document nesting beyond a single object's field ops, and depth > 1 (the shader save slot is already the `SdfFieldSave {distance, material, gradient}` struct, so the raise is one indexed-stack migration) — tracked in [sdf-backlog.md](sdf-backlog.md); evidence + decision history in git (`sdf-accumulator-plan.md`, retired 2026-07-09). |
| Stochastic cell jitter (domain scatter) | ✅ | `CellJitter` op (`SdfOp.CellJitter` = 22) tiles space like `Repeat`, then per cell displaces the following prototype by a hashed offset, with an optional hashed isometric tumble (rotation about the cell center) and an optional hashed material-variant recolor — one instruction scatters a jittered field from a single prototype. The hash is INTEGER-ONLY (`sdfPcg3d`, canonical PCG3D on the two's-complement cell index xored with the seed), so cell decisions are bit-identical across both DXC targets. The per-cell `round()` cell wall is a distance discontinuity the over-relaxed march would step clean over (eroding thin content), so `AnalyzeLipschitz` bakes a reach-independent boundary step clamp — the LogSphere twin, `sqrt((s/2+j/2)²+2j²)/m` sized by the in-cell margin — that holds the march conservative: free for loose scatter, tightening only when packed. The per-cell position offset takes a **noise flavor** (`SdfNoiseFlavor` on the Blend/`header.z` lane): White (independent PCG3D uniform — the default, byte-identical), Blue (fixed-point R3 low-discrepancy — a de-clumped, more even scatter, integer-deterministic; measured min-spacing 0.42→0.73 and 45→0 close pairs vs White), or Gaussian (central-limit — offsets cluster toward the cell centre). Authorable from JSON via `CellJitterOp.Flavor`. Proven by Post `world-cell-jitter` (cross-backend parity), `world-cell-jitter-solidity` (single-backend, coverage recovers ~163k→~247k px with the clamp), `world-cell-jitter-flavors` (Blue's integer path, cross-backend), and the `sdf-lipschitz` bake assert. |
| Wallpaper symmetry folds | ⚠️ | 16 of the 17 IUC wallpaper groups as one `WallpaperFold` op: square/rectangular lattices (P1–P4G) and the equilateral hex lattice (P3–P6M), every branch an exact isometry, with a parity-material stride (checker / hex 3-coloring recolors cells from the palette) and a per-sample symmetry LOD (distant lattices keep their copies, skip their in-cell folds). **`P4G` renders as `p4`** — its diamond fold commutes with the quarter-turn while the per-cell turn count anticommutes with a diagonal mirror, so no reflection survives (measured: 4-fold rotation, zero mirror classes — the same signature as `P4`). Recovering it is a redesign of the turn cocycle. `CMM` and `P6` carried the same class of defect (they emitted `pmg` and `p3`) and are fixed. Group identity is verified by direct point-group measurement, not by parity: Post `world-wallpaper` (maxΔ1 cross-backend on both lattice families) proves the two backends agree, NOT that the group is the one named. |
| Angular domain-repeat fold | ✅ | `RepeatPolar` op (`SdfOp.RepeatPolar` = 23) folds the plane perpendicular to a chosen axis (`SdfPolarAxis` X/Y/Z; Y = the XZ ground plane, the default) into `count` equal angular sectors — one authored prototype repeats rotationally around the axis (gears, wheels, rotunda columns, clock ticks, flower petals) from a single instruction. An optional `mirror` flag reflects each sector across its bisector (kaleidoscope symmetry); an optional `materialStride` recolors per sector. The fold is a rotation (+ optional reflection) — both isometries, so the field stays 1-Lipschitz (factor 1, no step clamp, exactly like `Repeat`/`WallpaperFold`). Builder: `SdfProgramBuilder.RepeatPolar(count, axis, mirror, materialStride)`. Proven by Post `world-repeat-polar` (cross-backend parity, Vulkan SPIR-V vs Direct3D 12 DXIL). |
| Arbitrary-plane reflection fold | ✅ | `SymmetryPlane` op (`SdfOp.SymmetryPlane` = 26) reflects everything on an ARBITRARY plane's negative side (`dot(p, normal) + offset < 0`) onto its positive side — the general-normal fold that REPLACED the axis-aligned `SymmetryX`/`SymmetryY`/`SymmetryZ` opcodes (ids 13–15, collapsed into this one; the builder keeps `SymmetryX/Y/Z()` as sugar that emit it — for `normal = x̂, offset = 0` it is `abs(p.x)` to the bit). One authored half repeats mirror-imaged across any plane (a kaleidoscope leaf, a bilateral body, the reflect atom of a KIFS fold) from a single instruction; the normal is normalized host-side. The fold is a reflection — an isometry, so the field stays 1-Lipschitz (factor 1, no step clamp, exactly like `WallpaperFold`/`RepeatPolar`). Builder: `SdfProgramBuilder.SymmetryPlane(normal, offset)`. Proven by Post `world-symmetry-plane` (cross-backend parity, Vulkan SPIR-V vs Direct3D 12 DXIL). |
| Sinusoidal surface relief (Displace) | ✅ | `Displace(frequency, amplitude)` (`SdfOp.Displace` = 24) is a FIELD op (like `Onion`/`Dilate`): `d += amplitude·sin(fx·x)sin(fy·y)sin(fz·z)`, a separable sine-product basis added straight to the accumulated field — the SDF-native height/parallax map, except the relief is REAL geometry (it shadows and self-occludes, unlike a texture-space trick). Deterministic float trig keeps it cross-backend parity-safe with no hashed noise table (richer fBm/gradient noise is deferred — it needs an integer-hash basis like `CellJitter`'s for cross-backend bit-stability). NOT 1-Lipschitz: the relief's gradient reaches `amplitude·‖frequency‖`, so `AnalyzeLipschitz` bakes a conservative `1 + amplitude·‖frequency‖` step clamp. Builder: `SdfProgramBuilder.Displace(frequency, amplitude)`. Proven by Post `world-displace` (cross-backend parity), `world-displace-solidity` (single-backend, the clamp holds the over-relaxed march), and a `sdf-lipschitz` stepScale assert. |
| Cross-coupled domain warp (DomainWarp) | ✅ | `DomainWarp(frequency, amplitude)` (`SdfOp.DomainWarp` = 25) is a POINT op (like the fold ops): `p += amplitude·(sin(fx·y), sin(fy·z), sin(fz·x))` before the wrapped shapes evaluate — organic bulge/wobble/terrain from a non-separable, cross-coupled sinusoid (each axis driven by the NEXT axis's coordinate). Deterministic float trig, same parity posture as `Displace` (richer noise likewise deferred). NOT an isometry: the same `1 + amplitude·‖frequency‖` metric-stretch clamp applies, and the point's max travel (`amplitude·√3`) folds into a downstream twist/bend's reach. Builder: `SdfProgramBuilder.DomainWarp(frequency, amplitude)`. Proven by Post `world-domain-warp` (cross-backend parity), `world-domain-warp-solidity` (single-backend, the clamp holds the march), and a `sdf-lipschitz` stepScale assert. |
| Material model | ✅ | Palette entries carry albedo + emissive (self-glow) + Blinn-Phong specular/shininess, two packed words per material with one shader decode point; an all-zero specular/emissive material reduces to pure lambert exactly. Proven by Post `world-menagerie`. |
| Diegetic screen glow + CRT | 🔶 | A booted stand's diegetic screen renders a CRT glass face (barrel curvature + rounded bezel + native-line scanlines + vignette + fresnel glint + bloom, in `sampleScreenSurface`) AND emits colored light into the room — its per-frame framebuffer average, via a binding-11 `sdfScreenLights` buffer summed with the sun in the world shade loop (up to 4 screen lights); a per-frame `AmbientScale`/`SunScale` dims the room so the glow dominates (the overworld mood). Additive: the default (no bound screen, env 1.0) renders the base image. CRT-face cross-backend parity is proven by Post `world-screen` (the binding-11 buffer is read every frame there); the CRT face + dimmed mood are confirmed in a live overworld capture. The lit-screen room glow is wired through the overworld; a fully-lit beauty shot needs game content (an external showcase ROM). |
| SDF engine host | ✅ | `SdfWorldEngine` (device-explicit core: beam cull → per-view render → split-screen composite over a viewport table of cameras + regions; submit-and-wait harness mode AND fire-and-forget node mode) + `SdfEngineNode` (the host-model `IRenderNode`), both in `src/Puck.SdfVm`, fully backend-neutral. Post drives the same engine its stages validate; the demo/overworld composites through the node. |
| Sphere-trace marcher (Lipschitz-safe) | ✅ | The per-pixel march is Keinert Enhanced Sphere Tracing — over-relaxed steps with a disjoint-sphere step-back + a footprint-adaptive hit threshold (resolution-independent silhouettes, ~2× fewer steps on loaded scenes). It rides a per-program **Lipschitz `stepScale`** (`SdfProgram.AnalyzeLipschitz`, baked into the packed words) so a non-1-Lipschitz warp (twist/bend/log-spherical) or an eccentric ellipsoid can't overstep and hole; isometric programs stay byte-identical. Lit normals are ANALYTIC by default (2026-07-09: the forward-mode gradient dual `mapGradCore`, one hit-only dual eval replacing the 4-tap tetrahedron probe — hero cross-backend parity improved 51→11 px, gpu-budget 1.87→1.58 ms, runtime A/B via `sdf.normals taps|analytic`, gated by Post `world-analytic-normal`); a tile-masked soft-shadow penumbra march grounds lit objects. **Shortlist wave 1 (landed 2026-07-08, from the SOTA survey's ranked rows):** the Bán 2023 auto-relaxed marcher (row 8) plus a four-bound gap teleport with an aggressive clear-space search (early-abandons after a stall limit, so a gap-less tile doesn't burn its full step budget); the soft shadow gained Aaltonen's closest-point-behind guard (row 7 — fixes a black-speck band where the parabola estimate put the closest point at/behind the current sample); iq's 5-tap normal-ladder ambient occlusion (row 1); curvature shading + gradient ink outlines (row 2, stylized, **off by default**); and the Tier-0 coverage AA (silhouette vs. sky) was corrected so solid surfaces stay solid. Verified by the full Post battery (the existing world/solidity/parity stages — no new gates needed, since these are shading/march refinements, not new ops) plus the `sdf` debug mode's scripted capture tour ([docs/examples/scripts/sdf-debug.console](examples/scripts/sdf-debug.console)). Remaining march/perf levers (segment tracing, a compute shadow-cull, tape pruning, noise ops) tracked in [sdf-backlog.md](sdf-backlog.md). |
| Dynamic transforms | ✅ | `TransformDynamic` reads per-frame slots from a GPU buffer, so moving entities update every frame **without** re-uploading the static scene program. Proven by the Post `dynamic-transform` stage. |
| GPU-driven culling + indirect dispatch | ✅ | Beam pass → cull-args reduction → GPU-written indirect views dispatch; sky margin is never dispatched. Both backends. Indirect *draw* is deferred (engine seam unchanged). |
| Instance capacity (`MaxInstances`) | ✅ | **16384** (raised 1024→4096 on 2026-07-08, 4096→16384 on 2026-07-09 when its measure-gate fired and was satisfied). The gate's story: `sdf.bench sweep` measured `sdf-beam` scaling linearly in instance count (O(instances), exponent ≈0.97, 187.8 ms beam @4096, ~750 ms extrapolated @16384); the mask-first uniform-grid instance cull (`sdf-instance-cull.comp` — a dedicated pre-pass computing each tile's instance mask via a deterministic CSR world-space grid, with the cone march consuming the tile-MASKED field; the discovery en route: the O(n) was the march's per-sample enumeration, never the binning) collapsed it to **6.6 ms @4096 / 21.8 ms @16384** beam+mask, leaving frames views-bound. The measured destruction budget rides this: ~1024 in-frame scattered carves at 60 fps, off-screen persistent carves ~free — full tables in [sdf-bench-notes.md](sdf-bench-notes.md). |

| Debug visualization | 🔶 | The split engine carries the RT cull-parity debug host (`sdf-world-rt-debug.rq.comp.hlsl`) and the demo's SDF debug view modes (`debug.view.*` console verbs / the Debug: Engine pad page): `off`/`depth`/`normals`/`raydir`/`material-id`/`iteration-count`, plus two 2026-07-08 additions — `termination` (why the march loop exited, per pixel, reconstructed post-loop) and `slice` (the IDEAL distance field as a cross-section, wall to wall — the deliberate termination/slice split, so the debug view distinguishes what the real march did from what the field actually contains). ## 2. Dual backend: Vulkan + Direct3D 12

The whole showcase runs on **either** backend through one neutral seam
(`Puck.Abstractions`). See [feature-parity-summary.md](feature-parity-summary.md)
for the parity verdict and [feature-parity-table.md](feature-parity-table.md)
for per-row provenance.

| Capability | Status | Notes |
|---|---|---|
| Single-source shaders | ✅ | One HLSL source compiled at build time by DXC to **both** SPIR-V (Vulkan) and DXIL (D3D12) — including the inline-ray-tracing kernels. `src/Puck.SdfVm/Puck.SdfVm.csproj` is the recipe; `ShaderModuleLoader` sniffs SPIR-V vs DXBC/DXIL containers. |
| Cross-backend zero-copy surface sharing | ✅ | Content produced on one API, presented by the other, no CPU round-trip — **both directions** GPU-verified: Vulkan host + D3D12 producer and D3D12 host + Vulkan producer. Devices are LUID-matched. Described as data by a `world` run document (`graph.produce`): a mismatched `produce` is rejected with an attributed error (the shared world renderer runs host-backend only, pending the cross-backend re-host); same-backend `world` documents run live. Live zero-copy proof: Post C1/C2. |
| Runtime backend hot-switch | ✅ | `BackendSwitcher` toggles Vulkan ↔ D3D12 live; presenter persistence proven by Post D4. |
| Cross-backend differential fuzzing | ✅ | Process-isolated fuzzer renders generated scenes on both backends and diffs. The generator spans all 7 primitives, all 7 blends, and the twist/bend/elongate warps. Default posture is RELAXED (mean/spread guards only — real divergences explode both; FP-noise classes pass); `PUCK_PARITY_STRICT=1` opts into the pixel-perfect delta-mass calibrations (the long-term ideal — see `ParityThresholds`). Run via the Post `fuzz` stage (`--stage fuzz --fuzz-seed <n>`) or `tools fuzz` for isolated sweeps; the `fuzzing` document section is a schema-validated parse target (the live path is the Post stage, not a document). |
| Mixed-backend split screen | 🔶 | Per-viewport composition (up to 5 regions) where the *content* device and *host* device are different APIs, plus hosted child surfaces per viewport (a `world` run document's `child` graph field; the Post `world-child` stage proves it). A literal "top half D3D12, bottom half Vulkan in one window" is a composition of these verified pieces, not itself a standing demo. |
| GPU perf counters | ✅ | Cross-backend timestamp counters: per-pass GPU-ms + share on both backends. Enable with `PUCK_TIMING=1`; regression tripwire is Post D1. |

## 3. Everything as data: the run document

| Capability | Status | Notes |
|---|---|---|
| `puck.run.v1` document | ✅ | One versioned JSON document (`PuckRunDocument` in `src/Puck.Scene`) is the source of truth for a run: `host` (window/backend/present mode), `scene` (materials + SDF objects), `viewports` (regions + content sources), `graph` (root node), `input` (controller routing), `validation`, `fuzzing`. Version discriminator is gated before anything binds; root carries forward-compat `Extensions`. |
| CLI ⇒ document collapse | ✅ | Every `Puck.Demo --*` flag synthesizes a run document (`DemoRunDocuments.Synthesize`) — there is exactly ONE code path. Flag-synthesis determinism is Post A4. |
| JSON Schema export | ✅ | `schema/run.schema.json`, regenerated via `Puck.Demo --emit-schema <path>`. Every example document in [docs/examples/](examples/) validates against it (the Post `run-document` stage). |
| Graph node kinds | ✅ | `$type` on `graph`: `world` (the document's scene + viewports rendered live through the shared `SdfWorldRenderBuilder` on the HOST backend; cross-backend `produce` and the `child` boolean are rejected with attributed errors) and `overworld` (the demo: a `consoles` list of gaming-brick sources seats the room's bootable stands; an optional `library` of cartridges). Both graph kinds flow through the ONE shared render assembly. |
| `--rom` direct boot + fourth-wall exits | ✅ | `Puck.Demo --rom <path> [--rom-exit "0xC000>=1"]` boots straight INTO the cartridge — the IMMERSED overworld: `DemoRunDocuments` synthesizes an `OverworldNode { immersed: true }` with one stand per potential player (the same ONE code path), and each connecting controller (up to 4) is auto-seated at (boots + takes over) its own machine, panes tiling the screen 1→2→3→4 as pads join. When ANY machine's `exit` condition holds (a `gaming-brick` source field: work-RAM address 0xC000–0xDFFF as an `"0x…"` string + op + value; polled after each stepped frame), the FOURTH WALL BREAKS: the panes ease away and the room is revealed — every active player standing at their stand, the games continuing on the diegetic screens (machines never reset). Without a handler (a plain `world`-document `exit`) the condition instead requests the same clean shutdown `host.exitAfterSeconds` uses. Gates: `--validate-overworld` (hash unchanged — seating/ownership is host-side routing), the Post `run-document` stage, the Humble `battery-save` stage. |

| Linux / Steam Deck hidraw transport | 🧪 | Deferred; `Puck.Input` core is OS-agnostic, only the Windows transport exists (`Puck.Platform`). |

## 10. Emulation: the GamingBricks (experimental)

Handheld emulators use the deadpan `<Tier>GamingBrick` naming: **Humble**
(GB/GBC), **Advanced** (GBA), Folding (DS, future). Both cores are standalone
experimental subsystems — no `src/Puck.*` references yet. The north star is
[ideal-gaming-brick-plan.md](ideal-gaming-brick-plan.md): a deterministic
cross-generation machine (three rules: deterministic; carry-forward; two
players on different generations stay bit-locked over a link cable).

| Capability | Status | Notes |
|---|---|---|
| Humble (GB/GBC) core | ✅ | SM83, unified 16-bit timer/DIV/serial counter, docboy-derived PPU, SameBoy-derived integer APU, full mapper set. **Whole-machine mid-frame snapshot/restore and `Fork()`** — the foundation for rewind, netplay, and link co-sim. Battery: `dotnet run --project experimental/Puck.HumbleGamingBrick.Post` (determinism / snapshot round-trip / battery-save / fork-determinism / blargg / mooneye). |
| Battery saves (persistent SRAM + RTC) | ✅ | A battery-backed cartridge's external RAM persists to `<romPath>.sav` — loaded at machine assembly (a power-on cartridge read, a deterministic config input), flushed debounced-on-dirty, forced on reboot/dispose — with the mapper's clock appended as a footer (MBC3: the standard 48-byte RTC layout; HuC3: a 16-byte minutes/days block, our own convention — no cross-emulator standard exists for it). The clock RESUMES where the last session left it (the footer's wall timestamp is stamped for foreign-emulator interop but ignored on load — time pauses while off, never goes backward, so Pokémon G/S's own elapsed-time bookkeeping stays consistent). The save TRAVELS with the cartridge across the debug demote/promote reboot (hardware cartridge-move semantics) and is deletable live via the Bricks debug page's "Clear save" verb. Seam gate: the Humble battery's `battery-save` stage. |
| Advanced (GBA) core | ✅ | ARM7TDMI + full bus/prefetch/DMA/timers/IRQ per-cycle model, full PPU + APU, Flash/SRAM/EEPROM + RTC. 38/38 AGS aging-cartridge cells with real BIOS; Pokémon Emerald boots. No snapshot API — determinism is register+framebuffer identity. Battery: `dotnet run --project experimental/Puck.AdvancedGamingBrick.Post` (+ a deep diagnostics suite: mGBA lockstep co-sim, cycle/state traces, render hashes). |
| Machine-fleet bench | ✅ | `--bench` in the Humble Post exe: fleet scaling in both shapes (independent streams / shared choir stream) 1T+MT to N=256, burst catch-up, `Create`/`Snapshot`/`Restore`/`Fork` latency+allocation, mailbox-check cycle, per-machine footprint — with built-in serial-vs-parallel and same-stream bit-lock guards (exit 1 on divergence). Measured plan: [docs/machine-fleet-plan.md](machine-fleet-plan.md). |
| Networked / multi-machine link play | 🧪 | **Humble in-process link play WORKS**: `SerialComponent` exchanges bits with a linked peer (correct internal/external clock semantics, per-transfer serial interrupts), and `SerialLinkSession` owns the deterministic instruction-interleaved pair-stepping (a linked pair is ONE step unit — the parallel-fleet-safe shape). Gated by the Humble battery's first Tier C stage (`serial-link`: dmg↔cgb byte exchange, replay-identical across runs); in the overworld, cabinets link/unlink live via the Bricks debug page's Link verb. Still open: the Advanced (GBA) side (`IGbaLink`/`NullGbaLink` stubs), the cross-generation bit-lock (Humble plan M5's golden-replay trace on a real link game), and anything cross-process/networked — untested in practice. |
| Emulator-as-content-source | ✅ | The overworld IS the bridge: `GamingBrickChildNode` hosts the Humble machine as a live pane (fixed full-frame GPU allocation, per-frame upload, `OverworldBrickTimeline` input lockstep) — a machine is a first-class world citizen. The GamingBricks are the deliberately chosen stand-ins for hosting a foreign engine: deterministic, self-verifying tenants whose alien clock, framebuffer format, input timing, and link-cable comms exercise the full hosting contract — with the tenant provably correct, every integration bug is attributable to the host. Still open: the GBA core as a pane (overworld panes run the SM83 costumes), zero-copy framebuffer upload (fleet-plan backlog). |

## 12. Supporting libraries

| Library | What it gives you |
